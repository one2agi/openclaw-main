# 微博匿名采集方案

**研究日期：** 2026-04-18  
**研究员：** 墨灵 🔍

---

## 是否有匿名方案：**有**

微博存在无需 Cookie、无需登录的匿名端点，可通过移动端 API 获取热搜数据。

---

## 核心限制

| 限制类型 | 说明 |
|---------|------|
| **非官方 API** | 热搜匿名端点为前端页面接口，非官方开放 API，随时可能失效 |
| **IP 频率限制** | 无认证请求有 IP 级别频率限制，约每分钟 5-10 次 |
| **数据字段有限** | 匿名端点返回热搜词 + 热度值，无详细讨论量/阅读量 |
| **无用户数据** | 无法匿名获取个人微博内容，需 Cookie 授权 |
| **容器 ID 依赖** | 热搜 containerid 路由参数可能随版本更新变化 |

---

## 关键 API 端点

### 1. 热搜榜单（匿名可用）⭐ 推荐

```
GET https://m.weibo.cn/api/container/getIndex?containerid=106003type%3D25%26t%3D3%26disable_hot%3D1%26filter_type%3Drealtimehot
```

**请求头（必须）：**
```
User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 14_0 like Mac OS X) AppleWebKit/605.1.15 (KHTML, like Gecko) Version/14.0 Mobile/15E148 Safari/604.1
Accept: application/json, text/plain
Referer: https://m.weibo.cn
```

**返回字段：** `desc`（热搜词）、`hot_scheme`（跳转链接）、`label`（标签）、`raw_hot`（原始热度）

**参考实现：**
- GitHub: `RusianHu/weibo_hotsearch_mcp` — Python asyncio 版本
- GitHub: `Xarrow/weibo-scraper` — 支持 `get_realtime_hotwords()` 方法

### 2. 热搜额外端点

```
# 实时热搜（备用）
GET https://m.weibo.cn/api/container/getIndex?containerid=231583&luicode=10000011

# 娱乐榜
GET https://m.weibo.cn/api/container/getIndex?containerid=231659&luicode=10000011
```

### 3. 第三方聚合 API（无需自建）

| 提供商 | 端点 | 限制 | 备注 |
|--------|------|------|------|
| **uapis.cn** | `https://uapis.cn/docs/api-reference/get-misc-hotboard` | 匿名可用，有 QPS 限制 | 聚合 40+ 平台热搜 |
| **xxapi.cn** | `https://xxapi.cn/doc/weibohot` | 免费 API | 实时热搜排行榜 |
| **天聚数行 TianAPI** | `https://www.tianapi.com/apiview/100` | 需注册，有配额 | 每 30 分钟更新热搜前 50 |

## 移动端 vs 网页端对比

| 对比维度 | 移动端 (m.weibo.cn) | 网页端 (weibo.com) |
|---------|---------------------|-------------------|
| **认证要求** | 无需 Cookie | 需要登录态 |
| **反爬强度** | ⭐ 较宽松 | ⭐⭐⭐⭐ 较严格 |
| **API 结构** | JSON REST 接口，易解析 | 大量 JS 渲染，需 Selenium |
| **频率限制** | IP + 请求头检测 | UA + Cookie + IP 三重 |
| **稳定性** | 接口可能随时变 | 相对稳定 |
| **数据完整性** | 基础热搜字段 | 更丰富但需登录 |
| **推荐程度** | ⭐⭐⭐⭐⭐ 优先使用 | 仅作为备选 |

**结论：移动端 API 风控更松**，无需任何认证即可访问热搜数据，是匿名采集的首选路径。

---

## 官方开放平台 API（补充信息）

微博官方开放平台 (`open.weibo.com`) 提供 REST API：

- **认证方式：** OAuth 2.0（必须）
- **免费额度：** 极低，个人开发者几乎不可用
- **热搜数据：** 官方 API **不提供**热搜榜单接口
- **请求限制：** 按 App 级别限流，付费版才有足够配额
- **评估：** 对于热搜采集，官方 API **不适用**

## 可行性评估

### 热搜榜单匿名采集：⭐⭐⭐⭐⭐

- ✅ 已有多个活跃 GitHub 项目验证可行
- ✅ 移动端 API 无需 Cookie，代码已开源
- ✅ 数据格式为 JSON，解析简单
- ⚠️ 属于非公开接口，有被封禁风险
- ⚠️ 高频请求会触发 IP 封禁

### 个人微博内容采集：⭐

- ❌ 几乎所有微博详情接口都需要登录 Cookie
- ❌ 官方 API 需要申请且配额极低
- ✅ 可通过移动端抓包找到部分公开微博接口（仍需处理登录态）

## 推荐方案

### 方案一：移动端 API 直采（热榜采集首选）⭐⭐⭐⭐⭐

```
Python + httpx + 移动端 UA
1. 请求 m.weibo.cn/api/container/getIndex 热搜端点
2. 添加 iPhone UA + Referer 头
3. 控制请求频率 < 5次/分钟/IP
4. 异常时自动重试 3 次
```

**优点：** 免费、无需 Cookie、稳定可用  
**缺点：** 非官方接口，需维护适配

### 方案二：第三方聚合 API（快速接入）

使用 `uapis.cn` 或 `xxapi.cn` 的免费接口。

**优点：** 开箱即用，无需抓包维护  
**缺点：** 依赖第三方稳定性，有调用限制

### 方案三：官方 API + 自建账号（高精度需求）

申请微博开放平台账号，配合 Cookie 池使用。

**优点：** 数据准确、合规  
**缺点：** 成本高、申请周期长

---

## 风险提示

1. **法律合规：** 微博服务条款禁止大规模自动化抓取，数据使用请确保符合《数据安全法》
2. **IP 保护：** 高频采集建议配合代理池，避免单 IP 被封
3. **接口失效：** 非官方端点随时可能更新，请定期测试
4. **数据质量：** 匿名端点仅提供热搜词和基础热度，无法获取评论数/转发数等深度指标
