## ⚡ 一句话结论

知乎 API 签名（x-zse-93/x-zse-96）是核心障碍，**推荐方案是 MediaCrawler（浏览器自动化方案）**，绕过签名获取，最新可用；进阶方案是 `lostjay/zhihu_android_crawler`（Unidbg 模拟），签名更稳定但配置复杂。

---

## 📊 核心结论

| 维度 | 结论 |
|------|------|
| x-zse-96 签名 | 2025年已从 x-zse-93 迁移到 x-zse-96；纯 Python/JS 补环境方案易失效 |
| 最可行方案 | MediaCrawler（Playwright 浏览器自动化，无需逆向） |
| Unidbg 模拟 | lostjay 方案（2025.07更新）可生成正确签名，但需 Java+Maven 环境 |
| 无签名端点 | 极有限，仅部分公开内容可用 |
| 反爬强度 | 高（验证码、IP 限频、Cookie 失效检测） |

## 一、当前可行的实现方案

### 方案1：MediaCrawler（推荐 · 浏览器自动化）

- **开源库/工具**：[NanmiCoder/MediaCrawler](https://github.com/NanmiCoder/MediaCrawler)
- **GitHub 链接**：https://github.com/NanmiCoder/MediaCrawler
- **Star 数**：47.6K（2026-04）
- **支持平台**：小红书、抖音、快手、B站、微博、贴吧、知乎 ✅

#### 实现思路

**核心原理：不逆签名，用浏览器环境"借"签名。**

1. 用 Playwright 启动浏览器，用户扫码登录知乎，保存 Cookie（登录态）
2. 每次请求时，通过 Playwright 在已登录的浏览器上下文中执行 JS 表达式，从知乎页面的 JS 运行时中获取签名参数（x-zse-96 等）
3. 用获取到的参数通过 httpx 发起真实 HTTP 请求
4. 数据存入 MySQL / SQLite / CSV / JSON 等

**优势**：无需逆向任何加密算法，大幅降低技术门槛。

#### 核心代码片段

```bash
# 安装（推荐 uv）
curl -LsSf https://astral.sh/uv/install.sh | sh
cd MediaCrawler && uv sync

# 安装浏览器驱动（标准 Playwright 模式）
uv run playwright install
```

**配置文件** `config/base_config.py`（知乎相关）：

```python
# 开启评论爬取

# 知乎关键词搜索
# python main.py --platform zhihu --lt qrcode --type search

# 知乎指定问题详情
# python main.py --platform zhihu --lt qrcode --type detail
```

**WebUI 方式（更简单）**：

```bash
# 启动 API 服务器（默认端口 8080）
uv run uvicorn api.main:app --port 8080 --reload
# 访问 http://localhost:8080 打开可视化界面
```

**CDP 模式（推荐，连接已有 Chrome）**：
- Chrome 地址栏输入 `chrome://inspect/#remote-debugging`
- 勾选 "Allow remote debugging for this browser instance"
- 程序复用已有登录态，降低风控

### 方案2：lostjay/zhihu_android_crawler（Unidbg 模拟）

- **开源库/工具**：[lostjay/zhihu_android_crawler](https://github.com/lostjay/zhihu_android_crawler)
- **GitHub 链接**：https://github.com/lostjay/zhihu_android_crawler
- **最新更新**：2025年7月23日
- **实现语言**：Java + Python

#### 实现思路

**核心原理：逆向知乎 Android App 的签名算法，用 Unidbg 在 Python 中模拟调用。**

1. 对知乎 Android App 进行逆向分析，提取签名相关的 native library
2. 使用 Unidbg 框架模拟 Android ART 虚拟机，加载 native library
3. 提供 RESTful API 服务器（Java），接收路径等参数，返回签名
4. Python 客户端调用 REST API 获取签名，组装请求头

#### 环境准备

```bash
# Java 11+
java -version

# Maven 3.6+
mvn -version

# Python 3.10+
python --version

# 克隆 Unidbg（推荐国内镜像）
git clone https://lostjay.xyz/gitea/github/unidbg.git
# 或 GitHub: git clone https://github.com/zhkl0228/unidbg.git

cd unidbg && mvn clean install && cd ..
```

#### 核心代码片段

```python
from zhihu_android.android import ZhihuAndroidCrawler

# 搜索内容

async with ZhihuAndroidCrawler() as crawler:
```

### 方案3：Playwright + LLM 解析（无签名方案）

适用于**不调用知乎 API，直接爬取页面内容**的场景。

#### 实现思路

1. Playwright 打开知乎问题页面，等待内容完全渲染
2. 页面截图或获取渲染后 HTML
3. 丢给 LLM（大模型）解析，提取结构化数据
4. 配置兜底：LLM 解析失败时使用 CSS/XPath 选择器

#### 核心代码片段

```python
import asyncio
from playwright.async_api import async_playwright, Page

async def fetch_zhihu_question(url: str) -> dict:
    async with async_playwright() as p:
        browser = await p.chromium.launch(
            headless=True,
            proxy={"server": "http://proxy:port", "username": "u", "password": "p"}
        )
        context = await browser.new_context(
            user_agent="Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 Chrome/121 Safari/537.36",
            locale="zh-CN"
        )
        # 注入脚本，隐藏 webdriver 标识
        await context.add_init_script(
            "Object.defineProperty(navigator, 'webdriver', {get: () => undefined});"
        )
        
        page = await context.new_page()
        await page.goto(url, timeout=60000)
        
        # 等待回答卡片稳定
        await page.wait_for_selector(".AnswerCard", timeout=15000)
        await asyncio.sleep(2)  # 等待异步内容加载
        
        # 方式1: 获取 HTML 给 LLM 解析
        html = await page.content()
        
        # 方式2: 备用 CSS 选择器
        title = await page.query_selector("h1.QuestionHeader-title")
        answers = await page.query_selector_all(".AnswerCard")
        
        return {"html": html, "title": title, "answer_count": len(answers)}
```

#### ⚠️ 局限性

- 需要解决**异步加载**：评论和回答是延迟注入的，需等待 DOM 稳定
- **风控严格**：频繁访问会触发验证码，需控制频率和复用 Cookie
- LLM 解析依赖 DOM 结构，页面改版后容易失效

## 二、x-zse-93 签名处理

### 签名算法最新情况

| 字段 | 说明 | 状态 |
|------|------|------|
| `x-zse-93` | 旧版签名标识，值为固定字符串 `101_3_3.0` | ✅ 固定不变 |
| `x-zse-96` | 新版签名（2023年后），由 Android native library 或 Web JS 生成 | ⚠️ 核心障碍 |
| `d_c0` | Cookie 中的签名，由 `/udid` 接口 set-cookie 获取 | ✅ 一次性获取后复用 |

**x-zse-96 签名生成链路**（逆向分析结果）：
```
请求路径 + 固定参数 → ed()函数 → tT.signature → 拼接"2.0_"前缀 → x-zse-96
```
- 涉及：知乎的 webpack 打包 JS、AES/DES 加密（Android端）
- Web 端通过 `window.__zhihu_staticsig__` 或类似方式暴露
- Android 端通过 native library（libzhihu.so）计算

### 开源签名库

| 库/项目 | 地址 | 说明 |
|--------|------|------|
| `lostjay/zhihu_android_crawler` | https://github.com/lostjay/zhihu_android_crawler | Unidbg 模拟 Android native，生成 x-zse-96，**2025年7月更新** |
| `tiam-bloom/zhihuQuestionAnswer` | https://github.com/tiam-bloom/zhihuQuestionAnswer | 包含 signature.py 示例，含 zhihuvmp.js 补丁 |
| JS 补环境方案 | https://zhuanlan.zhihu.com/p/638375048 | 纯 JS 补环境，不依赖二进制逆向 |
| 看雪论坛算法还原 | https://bbs.kanxue.com/thread-281041.htm | AES 查表法还原，纯 Python 实现 |

**⚠️ 重要提示**：
- x-zse-96 签名**不是静态的**，每次请求路径不同，签名就不同
- 知乎频繁更新签名算法，纯补环境方案**容易失效**
- 目前最稳定的方案是 **Playwright 浏览器自动化**（MediaCrawler）或 **Unidbg 模拟**（lostjay）

## 四、知乎反爬机制分析

### 1. 签名验证（x-zse-96）

- 所有 API 请求必须携带正确的 x-zse-96 签名
- 签名由请求路径 + Cookie + 客户端标识实时生成
- 无签名或签名错误 → HTTP 400/403

### 2. Cookie / 登录态检测

- 关键 Cookie：`d_c0`（从 `/udud` 接口获取）、`z_c0`（登录态）
- 登录态失效或被识别为非浏览器 → 请求被拒绝
- `navigator.webdriver` 被检测到 → 风控触发

### 3. 频率限制

- 短时间大量请求 → IP 被限流或封禁
- 验证码（滑块、人机校验）触发机制
- 建议请求间隔 ≥ 3 秒，配合代理池

### 4. UA 和 TLS 指纹

- 知乎会检测 TLS 握手特征
- 非主流 User-Agent 可能被标记
- 建议使用真实 Chrome UA

### 5. 2025-2026 新变化

- 签名从 `x-zse-93` 全面迁移到 `x-zse-96`
- Android App 签名算法更新，需要 Unidbg 重新适配
- Web 端 JS 代码持续混淆，浏览器自动化方案更稳健
- 风控更加严格，2025年9月后频繁访问触发验证码概率增加

## 五、无签名端点分析

**结论：几乎没有完全不需要签名的 API 端点。**

| 端点 | 是否需签名 | 说明 |
|------|-----------|------|
| `/api/v4/questions/...` | ❌ 必须签名 | 需要 x-zse-96 |
| `/api/v4/answers` | ❌ 必须签名 | 需要 x-zse-96 |
| `/question/` 页面 | ✅ 可直接爬 | 直接 HTML，限流和验证码 |
| RSS / sitemap | ⚠️ 部分可用 | 仅最新问题，数据有限 |
| 知乎开放平台 API | ❌ 需申请 | 官方渠道，需审核 |
