# 小红书匿名采集方案

**调研时间**：2026-04-18
**研究员**：墨灵

---

## 一句话结论

小红书**不存在真正意义上的匿名采集方案**——所有可用路径均依赖 Cookie 或需要逆向签名机制（xsec_token、x-mini-sig 等），本质上是用他人账号身份或破解手段来模拟请求。

---

## 是否有匿名方案

**无（完全匿名）**

> 所有已知路径都需要：Cookie、xsec_token、或其他形式的身份/设备签名。

---

## 核心限制

| 限制类型 | 说明 |
|---------|------|
| **Cookie 强依赖** | Web 端 API 需登录后的 Cookie（a1 字段等），APP 端需设备签名 |
| **xsec_token** | 笔记详情等敏感接口的防 CSRF token，需从特定接口（如 homefeed）获取，非可选 |
| **签名算法** | APP 端需 x-mini-sig、x-mini-gid、xy-common-params 等签名参数，由 Native ARM 代码生成 |
| **风控机制** | IP 频率限制、账号请求上限（约 10 万次后封号）、设备指纹检测 |
| **官方 API** | 需企业资质 + 审核，只开放部分接口，采集能力极为有限 |
| **法律风险** | 逆向签名、绕过风控可能违反《反不正当竞争法》及平台服务协议 |

## 关键 API 端点

### Web 端（需 Cookie）

| 接口 | 路径 | 说明 |
|------|------|------|
| 首页推荐 | `https://edith.xiaohongshu.com/api/sns/web/v1/homefeed` | 获取笔记 ID，xsec_token 来源 |
| 笔记详情 | `https://edith.xiaohongshu.com/api/sns/web/v1/feed` | 需 note_id、xsec_token（部分版本可选） |
| 笔记评论 | `https://edith.xiaohongshu.com/api/sns/web/v2/comment/page` | 需 note_id、cursor |
| 用户笔记列表 | `https://edith.xiaohongshu.com/api/sns/web/v1/user_posted` | 需用户 ID |

### APP 端（需签名，需 Cookie/设备信息）

通过 Rnote API（付费）等第三方服务代理，22+ 端点覆盖笔记、搜索、用户、评论等。

### 官方开放平台

- **平台地址**：`open.xiaohongshu.com`
- **可用接口**：笔记详情等有限接口
- **门槛**：需企业营业执照 + 审核

## xsec_token 的作用和必要性

### 是什么
`xsec_token` 是小红书 Web 端接口的防 CSRF 校验 token，用于验证请求来源合法性。

### 从哪来
- 从 `homefeed` 接口的响应中获取
- 由前端 JS 动态生成（需要逆向 JS 逻辑）

### 是否必须
| 接口 | xsec_token 必要性 |
|------|-------------------|
| homefeed（推荐流） | 不需要 |
| feed（笔记详情） | **部分版本必须**，实测某些版本可绕过 |
| comment（评论） | 不需要（只要有 Cookie） |
| search（搜索） | 不需要 |

### 绕过方案
xsec_token 的生成逻辑可被逆向，但：
1. 算法可能随时更新，维护成本高
2. 属于灰色地带，账号有被封风险

## 可行性评估

| 方案 | 匿名程度 | 稳定性 | 成本 | 合规性 | 评分 |
|------|---------|--------|------|--------|------|
| 官方开放平台 API | ⭐⭐⭐ 完全合规 | ⭐⭐⭐ 高 | ⭐ 需企业资质 | ⭐⭐⭐ 合规 | ⭐⭐ |
| 开源爬虫 + Cookie | ⭐ 无（必须 Cookie） | ⭐⭐ 中 | ⭐⭐⭐ 免费 | ⭐ 不合规 | ⭐⭐ |
| Rnote API（付费） | ⭐ 无（他们提供 Cookie 池） | ⭐⭐⭐ 高 | ⭐⭐ 按量计费 | ⭐ 不合规 | ⭐⭐⭐ |
| 逆向签名算法 | ⭐ 无 | ⭐⭐ 需持续维护 | ⭐⭐⭐ 免费 | ⭐ 不合规 | ⭐⭐ |

> ⚠️ **"匿名"严格来说不存在**——所有方案都要么用账号（Cookie）要么逆向算法，后者更是直接违反平台 TOS。

## 推荐方案

### 场景 1：企业级合规采集

**推荐：官方开放平台**
- 地址：`open.xiaohongshu.com`
- 要求：企业营业执照、备案网站、明确业务场景
- 适合：品牌舆情监控、内容分析（有限数据）

### 场景 2：舆情监控（可接受付费）

**推荐：Rnote API（rnote.dev）**
- 22+ 端点，稳定账号池，自动风控冷却
- 按量计费，失败不计费
- 需要申请 API Key，自带 Cookie 管理
- 适合：稳定的长期采集需求

### 场景 3：自建爬虫（开源方案）

**推荐：`2044145178/xhs`（GitHub）**
- Node.js 实现，支持 homefeed、笔记详情、评论
- 需要手动配置 Cookie（a1 字段 + 完整 Cookie）
- xsec_token 实测非必须
- 适合：有一定技术能力、临时性采集

### 场景 4：快速原型验证

**推荐：xhs Python 库**
- `pip install xhs`
- 需要 Cookie
- 适合快速验证想法，不适合大规模采集

## 局限性

- 数据时效性：**搜索结果可能在 2026-04 后有变化**，平台策略频繁更新
- 匿名性：**没有任何方案是真正匿名的**，都依赖账号或逆向
- 合规性：逆向签名绕过风控违反平台 TOS，存在法律风险
- 稳定性：所有非官方方案都面临算法更新导致失效的问题
