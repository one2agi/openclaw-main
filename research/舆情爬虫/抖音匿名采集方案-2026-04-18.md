# 抖音匿名采集方案
> 调研时间：2026-04-18 | 调研人：墨灵

---

## 是否有匿名方案：（有，但极其受限）

**答案：部分有，但无法实现稳定可扩展的匿名采集。**

---

## 核心限制

### 1. X-Gorgon / X-Bogus 签名机制
- 抖音（及 TikTok）几乎所有客户端 API 请求都携带 `X-Gorgon` 和 `X-Bogus` HTTP 头
- 这两个签名由抖音 App 内置的加密算法实时计算生成，每次请求不同
- **无签名 → 请求被拒绝（403 / 数据为空）**
- 要伪造签名，需要逆向工程抖音 App，这本身存在法律风险

### 2. 行为检测（Behavioral Fingerprinting）
- IP 声誉检测（数据中心 IP 直接封禁）
- TLS 指纹校验（模拟浏览器请求头不够）
- Cookie 链完整性检查
- 请求时序分析（机器人请求太规律）
- 无真实鼠标移动、视频播放等交互事件

### 3. 认证强关联
- 搜索 API：**需要 msToken / sessionid**
- 评论 API：**需要登录态**
- 排行榜/推荐 API：**需要更高权限的 Token**
- 即便是"公开可见"数据，平台也通过签名+频率限制来阻止批量获取

### 4. 法律合规风险
- 2025年9月环球律师事务所分析：即便数据"公开"，若平台施加技术限制（API限制），爬取仍可能构成不正当竞争
- 《网络安全法》《个人信息保护法》对自动化数据采集有明确约束
- 抖音用户协议明确禁止未经授权的自动化数据采集

## 关键 API 端点

### ✅ 网页版隐式数据（无需 Token，但不稳定）

抖音网页版在 HTML 中嵌入 `__UNIVERSAL_DATA_FOR_REHYDRATION__` JSON，可解析出部分视频/用户数据：

```python
import httpx
import json
from parsel import Selector

headers = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 Chrome/120.0.0.0 Safari/537.36",
    "Accept-Language": "zh-CN,zh;q=0.9",
}

# 视频详情页（嵌入了 __UNIVERSAL_DATA_FOR_REHYDRATION__ JSON）
async def scrape_video(url: str):
    async with httpx.AsyncClient(headers=headers, http2=True) as client:
        r = await client.get(url)
        sel = Selector(r.text)
        data_raw = sel.xpath("//script[@id='__UNIVERSAL_DATA_FOR_REHYDRATION__']/text()").get()
        if data_raw:
            data = json.loads(data_raw)
            # 解析 videoDetail 等字段
            return data
```

> ⚠️ 局限：只能获取基础元数据（标题、播放量、点赞数），**无法获取评论全文**，且容易被封

### ✅ 抖音开放平台官方 API（合规，但需要 Token）

文档：https://open.douyin.com/

**前提条件：**
1. 注册开发者账号
2. 创建应用（需审核）
3. 申请对应权限
4. OAuth2 授权获取 Access Token

**部分可用接口：**

```
# 视频搜索（需「内容理解」权限）
GET https://open.douyin.com/video/search
Headers: Authorization: Bearer <access_token>

# 用户信息（需「用户信息」权限）
GET https://open.douyin.com/user/info?open_id=<uid>

# 视频数据（需「视频数据」权限）
GET https://open.douyin.com/video/data?video_ids=<vid>
```

> ⚠️ **匿名性：零**。需要完整 OAuth2 流程，且数据仅限于自己账号或已授权用户

### ❌ 搜索 API
- **需要 msToken**（从登录态 cookie 中提取）
- **需要签名**（X-Gorgon）
- 无法匿名调用

### ❌ 排行榜/推荐 API
- **抖音热榜**（热点宝）：需商家账号
- **推荐 feed**：强依赖设备指纹+用户画像，无 Token 基本无法调用
- **搜索推荐词**：需登录

### ❌ 弹幕/直播间 API
- 直播间弹幕走 **WebSocket (WSS)** 协议
- 需要 room_id 和用户身份，**无法匿名**

## 可行性评估

| 采集目标 | 可行性 | 方案 | 稳定性 |
|----------|--------|------|--------|
| 单个视频基础元数据（播放/点赞/评论数） | ⭐⭐ | 网页版解析 | 低（频繁变动） |
| 视频完整信息（标题/描述/标签） | ⭐⭐ | 网页版解析 | 低 |
| 视频评论内容 | ⭐ | 浏览器模拟或第三方 | 中（高封号风险） |
| 搜索结果（关键词） | ⭐ | 第三方 API | 中（费用+限制） |
| 用户粉丝/关注列表 | ⭐ | 官方 API（需授权） | 高（合规） |
| 排行榜/推荐内容 | ⭐ | 无公开方案 | 极低 |
| 直播间弹幕 | ⭐ | WSS 模拟（技术门槛高） | 中 |
| 匿名采集（无需任何 Token） | ⭐⭐⭐ | 仅限基础公开信息 | 低 |

## 推荐方案

### 方案一：第三方聚合 API（适合快速启动）

**推荐：TikHub API**（覆盖抖音、TikTok、快手等 14+ 平台）
- 700+ 端点，无需自己逆向
- 需注册付费，无匿名性
- 适合舆情监控场景
- 文档：https://tikhub.io/

**备选：Apify Douyin Scraper**
- 无需登录即可抓取公开视频数据
- 按抓取量计费
- 有现成 actor 可直接调用

### 方案二：官方开放平台（适合长期合规运营）

**适合：企业级、需数据准确性、可接受 OAuth 授权流程**

1. 注册抖音开放平台：https://open.douyin.com/
2. 创建应用 → 申请权限（视频数据、用户数据等）
3. OAuth2 授权 → 获取 Access Token
4. 调用 API，定期刷新 Token

> 数据真实权威，但只能获取**已授权账号**的数据，无法做全局搜索/监控

### 方案三：浏览器模拟 + 代理池（技术门槛高，仅学术研究用）

- 技术栈：Playwright / Puppeteer + 住宅代理 + 随机 User-Agent + 行为模拟
- 可绕过部分反爬，但 IP 成本高、容易被识别
- 违反抖音服务条款，仅作技术研究参考

---

## ⚠️ 风险提示

1. **法律风险**：未经授权爬取抖音数据可能违反《反不正当竞争法》和平台服务协议
2. **封禁风险**：大规模爬取会导致 IP 被永久封禁
3. **数据质量**：非官方渠道数据存在缺失和延迟
4. **维护成本**：抖音频繁变更接口，爬虫维护成本极高
