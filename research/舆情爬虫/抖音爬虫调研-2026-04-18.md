# 抖音爬虫实现方案

> 研究时间：2026-04-18
> 研究者：墨灵 🔍

---

## ⚡ 一句话结论

**2025-2026年，抖音 Web 端爬虫的核心难点仍是 `a_bogus` 签名，但已有两条实际可落地的路径：①用 Playwright 复用浏览器登录态绕过签名（MediaCrawler 方案），②用 TikHub API 代生成签名（付费但省事）。纯 Python 补环境逆向 a_bogus 代价高、维护难，不推荐。**

**原因**：大量一手 GitHub 源码、官方文档、社区 issue 交叉验证。MediaCrawler 43K+ stars、Douyin_TikTok_Download_API 10K+ stars，是社区公认方案。

## 一、当前可行的实现方案

### 方案1：MediaCrawler（⭐ 推荐）

- **开源库/工具**：MediaCrawler
- **GitHub 链接**：https://github.com/NanmiCoder/MediaCrawler
- **Star 数**：43K+（截至 2026-04）
- **支持平台**：小红书、抖音、快手、B站、微博、贴吧、知乎

#### 实现思路

> **核心思路：不逆向 a_bogus，而是让浏览器帮你算。**

MediaCrawler 的核心创新是**用 Playwright 驱动真实浏览器，在浏览器上下文中执行 JS 获取 a_bogus 参数**，完全绕过了手动逆向的难题。

支持两种运行模式：

| 模式 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| **CDP 模式**（默认） | 连接已有的 Chrome 浏览器（版本≥144），复用已有的登录态、Cookie | 低风控、可复用登录态 | 需要本地有 Chrome |
| **标准 Playwright 模式** | 程序自己启动无头/有头浏览器 | 一键运行，无需配置 | 登录态需要重新维护 |

#### 核心代码片段

```python
# 进入项目目录
cd MediaCrawler

# 使用 uv 管理依赖（推荐）
uv sync

# 爬取抖音关键词搜索结果（含评论）
uv run main.py --platform douyin --lt qrcode --type search

# 爬取指定创作者主页作品
uv run main.py --platform douyin --lt qrcode --type detail

# 查看全部命令
uv run main.py --help
```

**配置示例（config/base_config.py）**：

```python
# 抖音相关配置
DOUYIN_KEYWORDS = ["舆情监控关键词1", "热点事件"]  # 搜索关键词
ENABLE_GET_COMMENTS = True  # 开启评论爬取
ENABLE_CDP_MODE = True  # 使用 CDP 模式连接已有 Chrome

# 数据存储（支持 CSV/JSON/JSONL/Excel/SQLite/MySQL）
DATA_STORE_TYPE = "jsonl"
```

**CDP 模式 Chrome 配置**：
1. 安装 Chrome（版本 ≥ 144）
2. 地址栏输入 `chrome://inspect/#remote-debugging`
3. 勾选 "Allow remote debugging for this browser instance"

### 方案2：Douyin_TikTok_Download_API（视频采集方向）

- **开源库/工具**：Douyin_TikTok_Download_API
- **GitHub 链接**：https://github.com/Evil0ctal/Douyin_TikTok_Download_API
- **Star 数**：10K+

#### 实现思路

内置了 **X-Bogus 和 A_Bogus 两种签名算法的实现**，支持视频解析、API调用、批量下载。

**⚠️ 重要限制**：必须提供有效的浏览器 Cookie 才能正常工作（需手动从 Chrome 登录抖音后提取）。

#### 核心代码片段

```python
# 安装 pip 包
pip install douyin-tiktok-scraper

from douyin_tiktok_scraper import DouyinAPI

api = DouyinAPI(cookie="你的抖音Cookie")

# 解析单个视频
result = await api.hybrid_parsing("https://www.douyin.com/video/xxx")

# 获取用户主页作品
user_data = await api.get_user_posts(sec_uid="用户sec_uid")
```

### 方案3：TikHub API（付费商业方案）

- **官网**：https://tikhub.io
- **特点**：700+ 接口，覆盖 14+ 平台

#### 实现思路

TikHub 是付费数据服务，提供抖音全量数据 API，无需自己处理 a_bogus 签名。提供 **a_bogus 签名生成专用接口**：

```
POST /api/v1/douyin/web/generate_a_bogus
```

```json
{
  "url": "https://www.douyin.com/aweme/v1/web/general/search/single/?...",
  "data": "",
  "user_agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36...",
  "index_0": 0,
  "index_1": 1,
  "index_2": 14
}
```

**费用**：每日签到可获取免费额度，注册送 $2 额度，按量计费。

### 方案4：Playwright 直接注入法（自建方案）

#### 实现思路

在 Playwright page context 中直接执行 JS 获取 a_bogus（MediaCrawler 的核心原理）：

```python
a_bogus = await page.evaluate("""
    () => {
        const params = "device_platform=webapp&aid=6383&keyword=关键词...";
        const post_data = "";
        const ua = navigator.userAgent;
        return window.bdms.init._v[2].p[42].apply(null, [0, 1, 8, params, post_data, ua]);
    }
""")
```

**优点**：无需逆向，完全依赖真实浏览器环境
**缺点**：依赖抖音页面 JS 结构，抖音更新可能失效

## 二、核心难点

### a_bogus 签名最新情况

| 项目 | 状态 | 说明 |
|------|------|------|
| x-bogus | ❌ **已废弃** | 抖音已全面替换为 a_bogus |
| 纯 Python 补环境逆向 | ⚠️ **可行但代价高** | JSVMP 混淆，维护成本极高，算法更新频繁 |
| Playwright 浏览器注入 | ✅ **当前主流** | MediaCrawler 方案，不逆向，靠真实浏览器算签名 |
| TikHub API 代生成 | ✅ **付费最省事** | TikHub 提供 a_bogus 生成接口 |
| 补环境开源方案 | ⚠️ **有人有但不公开** | MediaCrawler 作者曾考虑公开但最终未发布（见 Issue #297） |

**a_bogus 原理**：
- 由 URL 查询参数 + User-Agent 共同计算
- 采用 JSVMP 混淆技术
- 每隔一段时间算法会更新，需持续维护

### 其他限制

| 限制项 | 说明 | 应对方式 |
|--------|------|----------|
| **msToken** | Cookie 中的关键参数，有有效期 | 定期从浏览器刷新 Cookie |
| **s_v_web_id / verify_fp** | 设备标识参数 | TikHub API 可生成，或从浏览器获取 |
| **IP 频率限制** | 请求过快触发风控 | 加代理池，控制请求频率 |
| **账号风控** | 同一账号大量请求可能被限流 | 多账号轮换 + IP 代理 |

## 三、推荐实现步骤

### 路线A：零成本自建（MediaCrawler）⭐ 推荐

```
步骤1：安装依赖
  - Python 3.11+
  - Node.js ≥ 16.0.0
  - uv（Python 包管理工具）
  - Chrome 浏览器（版本 ≥ 144）

步骤2：克隆并配置 MediaCrawler
  git clone https://github.com/NanmiCoder/MediaCrawler.git
  cd MediaCrawler
  uv sync
  # 配置 config/base_config.py 中的 DOUYIN_KEYWORDS

步骤3：启动 Chrome CDP 调试
  - Chrome 地址栏输入 chrome://inspect/#remote-debugging
  - 勾选远程调试

步骤4：运行爬虫（二维码登录）
  uv run main.py --platform douyin --lt qrcode --type search

步骤5：定期维护
  - 观察 Cookie 是否过期
  - 关注 GitHub Issue 了解 a_bogus 更新情况
```

### 路线B：付费稳定方案（TikHub API）

```
步骤1：注册 TikHub 账号
  https://user.tikhub.io/users/signup?referral_code=1wRL8eQk

步骤2：获取 API Key

步骤3：使用 Python SDK
  pip install tikhub

步骤4：集成到舆情系统
  bogus = await tikhub.generate_a_bogus(url=api_url, user_agent=ua)
  results = await tikhub.douyin.search(keyword="舆情关键词")
```

## ⚠️ 局限性

- **Cookie 有效期不固定**：msToken 等参数需要定期维护
- **抖音算法可能随时更新**：a_bogus 更新后方案需同步更新
- **法律合规性**：请确保采集行为符合抖音服务条款和相关法律法规
- **付费方案依赖第三方**：TikHub 等服务存在可用性风险
- **数据规模受限**：大规模采集时风控风险显著上升
