🔐 OpenClaw 云服务器安全部署指南

📅 修订：2026-04-22 🖥️ 系统：Ubuntu 24.04 / Debian 12 🎯 目标：最小攻击面 + 受控访问

⚡ 核心结论

  • 端口 18789（Gateway）禁止暴露到公网
  • 所有外部访问通过 Nginx 反向代理 + SSL + Token 认证
  • 非 root 用户运行 + 防火墙 + fail2ban
  • CVE-2026-25253 敲响警钟：别心存侥幸

🏗️ 架构总览

[外部访问者] │ ▼ [域名解析 → SSL证书 → Nginx 443端口] │ │ │ 反向代理到本地 ▼ ▼ [Gateway Loopback 127.0.0.1:18789] │ ▼ [OpenClaw Agent + Skills + Cron]
✅ 核心原则：只有 HTTPS（443）对外暴露，Gateway 端口永远只监听 loopback。

📋 部署步骤

阶段1 服务器初始化

1.1 创建专用非 root 用户

apt update && apt upgrade -y adduser openclawops usermod -aG sudo openclawops mkdir -p /home/openclawops/.ssh cp ~/.ssh/authorized_keys /home/openclawops/.ssh/ chmod 700 /home/openclawops/.ssh chmod 600 /home/openclawops/.ssh/authorized_keys # 验证后断开 root 连接，用 openclawops 重新登录

1.2 防火墙（UFW）

sudo ufw default deny incoming sudo ufw default allow outgoing sudo ufw allow 22/tcp sudo ufw limit 22/tcp # SSH 限速防暴力 sudo ufw allow 80/tcp sudo ufw allow 443/tcp sudo ufw deny 18789/tcp # ⭐ 明确禁止 Gateway 暴露 sudo ufw enable sudo ufw status verbose

1.3 fail2ban 防暴力破解

sudo apt install fail2ban -y sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local # 编辑 jail.local，确保 [sshd] 启用 + maxretry=5 + bantime=3600 sudo systemctl enable fail2ban sudo systemctl start fail2ban

1.4 SSH 加固

编辑 /etc/ssh/sshd_config：禁止密码登录、禁止 root、Key 认证、建议换端口。

阶段2 安装 OpenClaw

2.1 安装 Node.js 24（最低 22.16）

curl -fsSL https://deb.nodesource.com/setup_24.x | sudo -E bash - sudo apt install nodejs -y node --version # 应输出 v24.x.x

2.2 全局安装 OpenClaw

npm install -g openclaw@latest openclaw --version openclaw onboard --install-daemon
⭐ 关键：向导中设置 Gateway Bind 为 loopback（不是 0.0.0.0）。
# 如果错过，手动设置： openclaw config set gateway.bind loopback openclaw gateway restart

2.3 systemd 服务配置

sudo loginctl enable-linger openclawops systemctl --user enable openclaw systemctl --user start openclaw journalctl --user -u openclaw -f # 查看日志

阶段3 Nginx 反向代理 + SSL

3.1 安装

sudo apt install nginx certbot python3-certbot-nginx -y

3.2 Nginx 配置

server { listen 80; server_name agent.yourdomain.com; # 替换为你的域名 location / { proxy_pass http://127.0.0.1:18789; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection "upgrade"; # ⭐ WebSocket 必须 proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $scheme; proxy_read_timeout 86400; } }

3.3 启用 + SSL

sudo ln -s /etc/nginx/sites-available/openclaw /etc/nginx/sites-enabled/ sudo nginx -t && sudo systemctl reload nginx sudo certbot --nginx -d agent.yourdomain.com sudo certbot renew --dry-run # 验证自动续期

阶段4 认证与权限加固

4.1 设置 Gateway Token

openssl rand -hex 32 # 生成强随机 Token openclaw config set gateway.token "粘贴Token" openclaw gateway restart

4.2 锁定凭证目录

chmod 700 ~/.openclaw/credentials/ chmod 600 ~/.openclaw/credentials/* chmod 700 ~/.openclaw/
⚠️ OpenClaw 将 API Key 明文存储在 ~/.openclaw/credentials/，必须锁死权限！

阶段5 验证

openclaw doctor # 健康检查 # 访问 https://agent.yourdomain.com 测试

🔐 安全检查清单

  • 非 root 用户运行 OpenClaw
  • SSH 使用 Key 登录，禁止密码
  • SSH 端口限速或更换端口
  • UFW 默认拒绝入站，开放 22/80/443
  • Gateway 端口（18789）被 UFW 拒绝
  • Gateway Bind 设置为 loopback
  • Nginx 反向代理 + SSL 生效
  • Gateway Token 已设置
  • fail2ban 运行中
  • 凭证目录权限 700

🚨 已知漏洞

CVE-2026-25253（2026 年初）

Censys 扫描发现 21,000+ 台暴露的 OpenClaw 实例。

漏洞成因：Gateway 默认绑定 0.0.0.0:18789，导致 WebSocket 劫持 RCE。

修复方案：始终使用 gateway.bind: loopback，通过 Nginx 代理访问。

🆘 紧急处理

发现异常访问？

sudo lastb # 查看失败登录 sudo fail2ban-client status # 查看 ban 情况 sudo ufw insert 1 deny from <恶意IP> # 手动封禁 journalctl --user -u openclaw -n 100 # 检查 OpenClaw 日志

Token 泄露？

openssl rand -hex 32 openclaw config set gateway.token "新token" openclaw gateway restart # 更新所有客户端配置
