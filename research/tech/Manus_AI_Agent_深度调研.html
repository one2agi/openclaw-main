3. 技术架构

3.1 多 Agent 协作系统

用户任务 ↓ ┌─────────────────┐ │ Planner Agent │ ← 分解任务为子任务序列 └────────┬────────┘ ↓ ┌─────────────────┐ │ Executor Agents │ ← 浏览器 / 代码 / 文件 / API 并行执行 └────────┬────────┘ ↓ ┌─────────────────┐ │ Verifier Agent │ ← 审查输出质量与正确性 └────────┬────────┘ ↓ 交付结果

3.2 底层模型

模型角色
Claude 3.5 Sonnet v1（Anthropic）主推理模型，核心任务执行
阿里巴巴 Qwen（微调版）辅助任务

首席科学家 Yichao "Peak" Ji 披露

3.3 Context Engineering（上下文工程）

Manus 团队的核心差异化技术，解决长程任务中的信息衰减问题：

  • 文件系统作为外部记忆：按需读写，而非将所有信息塞入 context window
  • 平均 50 次工具调用/任务，输入:输出 token 比 100:1
  • 架构迭代四次，团队将此过程戏称为 "Stochastic Graduate Descent"

3.4 云端沙箱执行环境

通过 E2B（安全云代码执行平台）为每个任务提供隔离的云端虚拟机，包含完整 Ubuntu 文件系统，Agent 可执行终端命令并安全运行不可信代码。用户可实时观看 Agent 的执行过程。

3.5 Agent Skills 开放标准

2025 年中支持 Agent Skills 开放标准，SKILL.md 定义技能规范，Manus 可读取目录、解析规范并执行 Python/Bash 脚本，实现专业化能力扩展。

5. 局限性 ⚠️

🔐 安全漏洞（SilentBridge，已修复）

  • 间接提示注入：网页/文档中的隐藏指令被 Agent 静默吸收并执行
  • Gmail 数据外泄：通过邮件连接器渗透并窃取敏感内容
  • 密钥泄露：内部 API 密钥、端点信息被提取
  • 远程代码执行：在沙箱环境建立 root shell
  • 跨租户访问：通过公开 CDN 未授权访问其他客户文件

来源：Aurascape 安全研究，2026 年负责任披露，Manus 已修复

⚠️ 可靠性问题

  • Beta 阶段 bugs 和不稳定
  • 复杂任务可能失败或耗时过长
  • 幻觉风险（验证 Agent 无法完全消除）
  • 上下文窗口限制

🚧 访问与成本壁垒

  • 邀请制访问限制
  • 高级定价 $39-$200/月
  • 企业安全/治理功能早期有限
  • 依赖第三方 LLM API，成本高
