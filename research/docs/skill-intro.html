Self-Improvement Loop

让 AI 记住你的纠正，不再重复犯错 · 自动检测模式 · 持续变聪明

Situation · 现状
AI 会重复犯错
  • 同样的逻辑错误说了好几遍
  • 纠正后换了方式，但过几天 AI 又忘了
  • 每次新对话都是"从头开始"
Complication · 问题
纠正无法积累
  • 纠正只存在于当前对话里
  • 没有系统自动记录和整理
  • 跨对话无法形成学习闭环
Question · 疑问
如何让 AI 持续改进？
  • 纠正能自动记录吗？
  • 重复的错误能自动发现吗？
  • AI 能主动学习和优化吗？
Answer · 答案
Self-Improvement Loop
  • 实时捕获纠正 → 自动写入笔记
  • 定时检测模式 → 发现重复问题
  • 用户决策 A/B/C/D → AI 改进执行

Features · 核心功能

三大核心能力

⚡

自动捕捉

监听器自动识别你说"不对"、"错了"等纠正，中英文都支持。不需要手动整理，说完就自动记。

自动监听 关键词检测 多语言
🔍

模式检测

每小时自动检查笔记，发现同一个问题出现 2 次以上就标记为"需要关注"。

自动整理 重复检测 阈值报警
📱

及时通知

通过聊天机器人发送提醒，你只需回复 A/B/C/D 决定怎么处理，不用手动操作。

独立通知 Telegram/Discord 简单决策
Decision · 你的决策

A/B/C/D 四种处理方式

机器人通知后，你只需回复 A/B/C/D 一个字母。这只是粗决策——选择后会有 skill-improvement 诊断 + 让你细选要修什么问题（有预览、有确认）。

A
创建新技能
这是个全新的能力需求
当这个问题意味着需要创建一个全新的技能来处理这类事情。
发生了什么
  1. 读取问题的原始记录
  2. 调用 skill-creator 创建技能
  3. skill-creator 会询问你具体需求、生成草稿
  4. 你预览草稿、确认后保存
  5. 更新笔记状态为"已解决"
  6. 删除待处理通知
✓ AI 自动生成新技能来处理这类问题
B
优化现有技能
某个技能需要改进
当这个问题指出某个现有技能不够完善，需要补充功能。
发生了什么
  1. AI 理解要优化哪个技能
  2. 调用 skill-improvement 诊断现有技能
  3. skill-improvement 展示问题列表
  4. 你勾选要修复的问题、有预览、有确认
  5. 更新笔记状态为"已解决"
  6. 删除待处理通知
✓ AI 自动改进现有技能
C
跳过
这次不需要处理
当这只是误报或一次性错误，不需要任何处理动作。
发生了什么
  1. 标记这个问题为"暂时忽略"
  2. 设置"已通知"状态
  3. 不再提醒这类问题
  4. 删除待处理通知
○ 静默忽略，不再打扰你
D
升华
这是个通用规则
当这个问题揭示了一个通用道理，需要写入 AI 的基础配置。
发生了什么
  1. 把问题提炼成简洁规则
  2. 根据类型写入对应配置文件
  3. 更新笔记状态为"已升华"
  4. 删除待处理通知

Customize · 自定义

添加你的专属触发词

每个人都有自己习惯的纠正方式，添加你的口头禅让捕捉更精准

📁 文件位置
hooks/handler.js
找到 CORRECTION_KEYWORDS 数组
✨ 示例
// 你的口头禅 "好像不对", "等等再想想", "不是我想的",
添加方法
1. 打开 hooks/handler.js
2. 找到 CORRECTION_KEYWORDS 数组（约第 88 行）
3. 在数组中添加你的口头禅，用逗号分隔
4. 保存文件即可生效

Reference · 参考资料

术语通俗解释

🤖 Agent（智能助手）
一个专门做某类任务的 AI。比如"代码助手"专门写代码，"项目经理"专门协调任务。每个助手独立学习，互不干扰。
📝 Learnings（学习笔记）
AI 的"错题本"。记录你纠正了什么、哪里出错了、需要什么新功能。相当于 AI 的笔记本。
🪝 Hook（监听器）
一个自动捕捉纠正的"雷达"。每当你说"不对"、"错了"，它就自动记下来，不需要你手动整理。
⏰ Cron（定时检查）
每隔一段时间自动运行的"巡逻员"。比如每小时检查一次学习笔记，发现重复犯错就提醒你。
🔍 Pattern（模式）
重复出现的问题。如果同一个错误出现了 2 次以上，说明这是个"模式"，需要认真对待。
📱 Bot（通知机器人）
通过 Telegram/Discord 等聊天工具发送提醒的机器人。它会告诉你"这个错误出现好几次了，要不要处理？"
🔄 Distill（提炼）
把零散的笔记整理成结构化的"问题清单"。把"你说的各种话"归类成"这个问题出现了几次"。
🧠 SOUL/AGENTS/TOOLS.md
AI 的"基础配置"文件。SOUL = 性格，AGENTS = 工作方式，TOOLS = 工具使用技巧。把重要规则写入这些文件。
