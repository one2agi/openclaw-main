## 一、OpenClaw Skill 技术架构

### 1.1 核心文件结构

一个skill本质是一个文件夹，最简结构：

```
my-skill/
├── SKILL.md           # 必需：YAML frontmatter + Markdown说明
├── claw.json           # 可选：清单文件（权限、入口点、版本）
├── src/                # 可选：JS/TS辅助代码
│   ├── index.ts
│   └── utils.ts
├── examples/           # 可选：使用示例
└── README.md           # 可选：人类可读文档
```

### 1.2 SKILL.md 格式（核心技术）

SKILL.md由两部分组成：

**YAML Frontmatter**（触发条件）
```yaml
---
name: weather-reporter          # 斜杠命令标识符，全小写+连字符
description: 获取当前天气和任意位置的预报。
# 当用户询问天气、要求天气预报时触发
# 也用于旅行规划或穿衣建议场景
context: fork                   # 可选：fork=subagent独立上下文，inline=共享上下文
disable-model-invocation: false  # 可选：true=仅用户手动触发，agent不自动加载
user-invocable: true            # 可选：false=仅agent可加载
---
```

**Markdown说明**（执行指令）
```markdown
# Weather Reporter

你是一个天气助手。当用户询问天气时：

1. 获取用户指定位置的当前天气
2. 展示2-3句话简洁摘要
3. 包含来源和发布时间
4. 询问是否需要深入某个话题

[输出格式规范]
[边界情况处理]
```

### 1.3 Skill vs Plugin 的核心区别

| 维度 | Skill | Plugin |
|------|-------|--------|
| **本质** | 文本指令集（Prompt+元数据） | 可执行代码（执行系统操作） |
| **开发语言** | Markdown + 可选JS/TS | 需完整代码实现 |
| **开发门槛** | **极低**：会写Prompt即可 | 高：需系统级编程 |
| **维护成本** | 低（纯文本） | 高（版本、依赖、安全） |
| **能力边界** | 教agent"知道什么/怎么做" | 扩展agent"能做什么" |
| **使用场景** | 知识型、流程型能力 | 系统集成、工具访问 |
| **ClawHub发布** | 直接上传 | 需安全审核（VirusTotal） |

> 简单说：**Skill改变agent知道什么/如何行为，Plugin改变OpenClaw能做什么**（新代码/工具）

### 1.4 Skill能做什么

- **网页抓取**：通过HTTP请求抓取数据
- **API调用**：REST/GraphQL/CLI工具包装
- **文件处理**：文本、PDF、图片等
- **工作流编排**：多步骤自动化流程
- **数据处理**：清洗、转换、分析
- **实时事件**：Webhook处理（Stripe/GitHub/Slack）
- **跨平台集成**：连接各类SaaS API

### 1.5 开发一个基础skill需要多少时间

| 复杂度 | 时间 | 示例 |
|--------|------|------|
| 简单（纯指令型） | **15-30分钟** | 邮件摘要、翻译、内容改写 |
| 中等（需API集成） | **1-3小时** | 天气查询、新闻聚合、竞品监控 |
| 复杂（多步工作流） | **4-8小时** | Shopify库存管理、房产线索跟进 |
| 企业级（安全+测试） | **1-3天** | 金融合规审查、医疗数据处理 |

**核心优势**：无需编译、无需容器、即写即测。平均构建时间约30分钟（数字Applied数据）。

## 二、Skill vs Plugin vs MCP vs ChatGPT Plugins 对比

### 2.1 横向对比

| 维度 | OpenClaw Skill | MCP Server | ChatGPT Plugin | Zapier Integration |
|------|----------------|------------|----------------|--------------------|
| **标准化程度** | AgentSkills标准（跨平台兼容） | Anthropic官方协议 | OpenAI专属（已停止新发布） | 私有协议 |
| **生态规模** | 3000+ skills（ClawHub） | 快速扩张中 | 基本废弃 | 6000+ app集成 |
| **开发门槛** | **极低**（Markdown） | 中等（TypeScript/SDK） | 中等 | 低（可视化配置） |
| **变现能力** | 中等（平台不成熟） | 高（协议层需求） | 低（平台关闭） | 低（平台方主导） |
| **分发渠道** | ClawHub / 私仓 | npm / GitHub | 停更 | Zapier市场 |
| **货币化模式** | 一次性销售/免费+增值 | 开源+付费托管 | 停更 | 平台抽成 |
| **市场规模** | 早期（但增速快） | 快速成长 | 衰退 | $1B+ |
| **目标用户** | 开发者/技术用户 | 开发者 | 非技术用户 | 非技术用户 |

### 2.2 关键结论

- **MCP是当前最被看好的协议标准**，Anthropic背书，大厂跟进，但变现路径仍在探索
- **ChatGPT Plugin已事实死亡**（2024年OpenAI关闭插件商店）
- **Skill的门槛最低**，但变现依赖ClawHub生态成熟度
- **Zapier的集成商变现被平台压得很低**——独立开发者难以靠集成本身获利

## 五、变现技术细节

### 5.1 ClawHub结算现状

**关键发现**：ClawHub截至2026年初**尚未内置支付系统**，开发者通过以下方式变现：

1. **SaaS引流模式**（最流行）
   - 发布免费skill连接到自己的SaaS API
   - 免费层处理基础查询
   - 高级功能需要付费API key
   - 例：天气skill → 连接到自己的天气数据服务

2. **portfolio+定制开发**
   - 将skill作为能力展示
   - 引导客户联系定制开发
   - 企业功能和支持合同
   - **企业定制：$500-$2,000/项目**

3. **外部支付桥接**
   - Gumroad/Patreon销售skill包
   - 手动处理购买和交付
   - 适合一次性买断型产品

4. **订阅模式**
   - 定期付费获取更新+支持
   - 社区访问权限搭售
   - 需自行管理订阅（Stripe/Paddle）

### 5.2 税务考量

- **美国开发者**：需报告所有收入（ClawHub不代扣税）
- **国际开发者**：取决于本地税法
- **SaaS引流收入**：可能需要征收销售税/增值税
- **建议**：收入超过阈值后咨询专业税务顾问

### 5.3 防盗版策略

| 策略 | 说明 |
|------|------|
| **代码混淆禁止** | ClawHub明确禁止代码混淆发布 |
| **VirusTotal扫描** | 所有代码发布前强制扫描 |
| **最小权限原则** | 只请求实际需要的权限 |
| **环境变量密钥** | 禁止硬编码凭证 |
| **渐进式功能** | 核心价值在服务而非代码 |
| **社区验证** | Verified badge建立信任 |

**核心逻辑**：由于skill本质是文本文件，完全防复制几乎不可能。护城河在于：
1. 服务层（API）+ skill组合
2. 先发优势+口碑积累
3. 持续更新和支持
4. 深度领域专业知识

## 七、行动建议

### 7.1 开发者快速启动路径

```
第1周：构建1个简单skill → 发布ClawHub免费版 → 收集反馈
第2周：迭代至可用状态 → 添加文档和示例
第3-4周：构建付费版本 → 通过Gumroad/Patreon销售
第2个月：扩展skill组合 → 建立领域专家定位
```

### 7.2 变现组合策略

| 收入来源 | 比例建议 | 说明 |
|----------|----------|------|
| ClawHub技能销售 | 40% | 被动收入，建立分发 |
| SaaS API引流 | 30% | 持续订阅收入 |
| 企业定制开发 | 20% | 高单价，高接触 |
| 教程/模板包 | 10% | 低边际成本 |

### 7.3 技术栈建议

- **Skill核心**：SKILL.md + 少量JS/TS辅助脚本
- **API服务**：自建或第三方（减少维护负担）
- **支付**：Gumroad（简单）/Paddle（更专业）
- **监控**：OpenClaw内置日志 + 自定义错误追踪
