## 一、项目概述

### 1.1 目标

在闲鱼建立AI提示词包虚拟商品店铺，通过双Agent协作实现半自动运营闭环。

### 1.2 双Agent核心定位

| Agent | ID | 角色定位 | 核心任务 |
|-------|-----|---------|---------|
| **无极** | `wuji` | 资源整合专家 + 质量管控 | 抓取→整理→上传→生成商品信息 |
| **星辰** | `xingchen` | 运营执行专家 | 上架→回复→记录销售数据 |

### 1.3 量化目标（暂定）

| 指标 | 目标 |
|------|------|
| 稳定在架商品数 | 50-100个 |
| 每日上架上限 | 5个（防风控） |
| 单月收益预期 | ¥500-3000 |
| 自动回复覆盖 | 100%买家咨询 |

## 五、协作流程

```
① 用户 → 下达任务给无极（资源链接/关键词/数量要求）
   ↓
② 无极 → 抓取资源（git clone / wget / curl）
   ↓
③ 无极 → 内容解析 + 四分类判断
   ↓
④ 无极 → 文件重命名 + 打包zip
   ↓
⑤ 无极 → 调用kuake-cli上传夸克网盘
   ↓
⑥ 无极 → 生成商品信息.md（含标题/描述/标签/夸克链接/建议定价）
   ↓
⑦ 无极 → 写入「待上架清单.md」，状态标记为「待上架」
   ↓
⑧ 星辰 → 轮询/读取「待上架清单.md」
   ↓
⑨ 星辰 → 按SOP执行上架（标题/描述/标签已固化）
   ↓
⑩ 星辰 → 执行闲鱼上架操作
   ↓
⑪ 星辰 → 更新「已上架商品清单.md」，记录商品链接/日期
   ↓
⑫ 星辰 → 监控xianyu-agent自动回复情况
   ↓
⑬ 星辰 → 定期反馈销售数据（咨询量/成交量）
   ↓
⑭ 无极 → 分析数据，提出优化建议（调价/下架/品类调整）
```

```

### 6.2 商品描述模板（暂定）

```
【痛点切入】
还在为[具体问题]烦恼？AI提示词帮你一键搞定！

【商品说明】
本资料包收录[N]条[模型名][场景]专用提示词，
覆盖[场景1/场景2/场景3]等多种用途，

【适用人群】
- 电商从业者/设计师/运营人员

【购买须知】
- 虚拟商品，拍下秒发货

- 收到后请及时保存，链接永久有效

## 十一、异常处理约定

| 情况 | 处理方式 |
|------|---------|
| 星辰发现资料包有问题（文件损坏/缺链接） | 写入「已上架商品清单.md」备注栏，标记异常，等无极处理 |
| 星辰遇到违禁词不确定 | 跳过该商品，在清单备注栏标注「待确认」，继续处理其他 |
| 无极发现商品信息不足 | 在商品信息.md中补充说明，通知星辰重新上架 |
| 闲鱼限流/封号 | 星辰立即上报，无极暂停新任务，排查后继续 |
