# 营销技能框架（基于 faiz 提供的 AIDA 变体）

## 核心框架

### 1. 钩子定位（1句）
**公式**：给谁 + 解决什么痛点 + 交付什么结果

- "这不是 Notion 模板，这是让你从'知道但做不到'变成'每天都做得出来'的个人行动系统。"
- "专治自律靠意志：把行动、复盘、奖励做成闭环，你只要跟着跑。"

### 2. 戳痛点（2-3句，具体到场景）

泛泛的痛点没用，要说具体场景：
- "收藏了一堆方法论，执行为零。"
- "每天忙，但说不清到底推进了什么。"
- "计划做得很满，三天后就崩盘。"
- "任务散落各处：备忘录/微信/脑子里，越记越焦虑。"

### 3. 亮结果（3-4句，可感知的变化）

把功能翻译成"用户会看到的变化"：
- "打开首页就知道今天做什么（不用想、不卡顿）"
- "一眼看见逾期、排期错误、进行中（决策成本更低）"
- "每天自动沉淀复盘与状态趋势（不靠记忆）"
- "做事有反馈、有奖励（坚持更容易）"

### 4. 给机制（3-5个独特机制）

- **控制中心式首页**：今日行动 + 行动规划，一屏进入执行
- **官方帮助中心**：新手5分钟上路，进阶有完整玩法，不怕买了吃灰
- **复盘 + 状况视图**：系统会校正你，而不是让你越用越乱
- **游戏化奖励**：让自律变成可反馈的闭环，而不是硬扛
- **想法实验室**：碎片先发酵，成熟才进执行流，不让半成品浪费执行资源

### 5. 降低门槛（提前解答3个顾虑）

- "我不会用 Notion 怎么办？" → 有帮助中心/路线图，照做即可
- "我很忙，维护成本会不会很高？" → 入口集中，日用只在首页跑
- "我之前买模板都吃灰" → 强调"执行闭环 + 奖励反馈 + 复盘校正"，不是资料库

### 6. 给证据（前后对比）

- "以前是记一堆，后来是每天推进1-3件关键事。"
- "从收件箱到行动规划到复盘，全链路不丢球。"

### 7. 明确行动（立刻能做的上手路径）

"第一天只做三件事：把待办收进来 → 选出今日行动 → 晚上用复盘区收尾。跑三天你就会有感觉。"

## 产品核心亮点（知行合一专属）

### 最有价值的5个点（内部代号：五把刀）

1. **控制中心式首页**：不是信息仓库，是仪表盘；一屏覆盖今日行动+规划+复盘+状况视图
2. **今日行动=数据库视图**：筛选逻辑自动刷新，不需手动维护待办清单
3. **游戏化奖励闭环**：XP/等级/奖励商店，解决"知而不行"，不是加几个emoji
4. **五级复盘闭环**：日志→周度→月度→季度→年度，系统自动校正方向，不靠自律
5. **想法实验室**：不成熟想法的等候室，前置过滤器防止执行资源浪费

### 核心设计哲学

> 知识系统和行动系统正交，不互相干扰。  
> 知识侧 = 积累层；行动侧 = 执行引擎。  
> 大多数模板是"硬盘"——存储导向；知行合一是"操作系统"——行动是默认输出。

## 写作禁忌

- ❌ 不要念功能列表（用户不关心有什么功能，关心能改变什么）
- ❌ 不要泛泛说"提升效率"（每个模板都这么说）
- ❌ 不要只讲正面（痛点要先说透，信任感来自"你懂我"）
- ❌ 不要忘记行动号召（让用户知道下一步做什么）

## 使用方法

在每次需要写营销内容前，先问自己7个问题：
1. 我的用户此刻正在经历什么具体场景？
2. 他们尝试过什么，为什么失败了？
3. 他们用了这个系统后，第三天会看到什么变化？
4. 这个系统最独特的机制是哪3个？
5. 用户最担心什么？
6. 用一句话描述前后对比。
7. 用户的第一个具体动作是什么？

## 经典文案框架（补充）

### AIDA 框架
**Attention → Interest → Desire → Action**

1. **Attention（注意力）**：用暴论/问题/数据钩住
2. **Interest（兴趣）**：展开痛点，让用户代入
3. **Desire（欲望）**：展示结果，让用户"看到"改变后的自己
4. **Action（行动）**：明确的 CTA

### PAS 框架（最适合痛点类文案）
**Problem → Agitate → Solution**

1. **Problem**：说出用户的具体处境（不要泛泛而谈）
2. **Agitate**：放大痛点，让用户感觉"这说的就是我"
3. **Solution**：引出产品作为答案

关键：PAS 的核心是**Agitate（放大）**，让用户感觉"尼玛，这正是我的感受"之后才给解法。

### FAB 法则（说服用内容结构）
**Feature → Advantage → Benefit**

- **Feature**：这个产品"有什么"
- **Advantage**："为什么"比别的好
- **Benefit**：用户"能得到什么"

> 大多数文案的错误：只写了 F，忘了 A 和 B。用户不关心你有什么功能，用户关心自己能得到什么。

### 四象限文案矩阵

| | 理性文案 | 感性文案 |
|--|---------|---------|
| **强逻辑** | FAB、技术白皮书 | 痛点故事、个人经历 |
| **弱逻辑** | 清单/对比表 | 情绪共鸣、身份认同 |

知行合一模板的文案适合：强逻辑 + 感性（理性讲机制，感性讲痛点共鸣）

## 文案自检清单（每次写完检查）

写完任何营销内容后，用这个清单过一遍：

**开头**：
- [ ] 有没有暴论/痛点句？前3秒能不能留住人？
- [ ] 用户能不能在第一句话里找到自己？

**内容**：
- [ ] 有没有具体场景（不是"提升效率"这种废话）？
- [ ] 有没有说结果/变化（不是讲功能）？
- [ ] 有没有3个以内的核心机制？

**结尾**：
- [ ] 有没有解答1-2个最常见的顾虑？
- [ ] 有没有明确的行动路径或 CTA？
- [ ] 有没有让用户感觉"我现在就能做"？

**禁忌检查**：
- [ ] 没有"提升效率"（泛泛而谈）
- [ ] 没有念功能列表（用户关心变化，不是功能）
- [ ] 没有"适合所有人"（缩小定位才有信任感）
- [ ] 没有忘了 CTA（每个文案都应该有下一步）

## MoltBook 专属写作指南

MoltBook 受众特征：
- 高思想密度，不喜欢广告味太重
- 喜欢"反直觉"观点，不喜欢鸡汤
- agent 视角：关注"系统 vs 工具"的哲学差异
- 最受欢迎的帖子 = 有洞察 + 有结构 + 有观点

**适合在 MoltBook 讲的切入点**：
- "知行合一 vs 其他模板的本质差异：不是功能，是架构哲学"
- "知识系统和行动系统为什么必须正交"
- "为什么大多数个人操作系统其实是硬盘"
- "想法实验室：大多数系统跳过的前置过滤器"

**不适合的写法**：
- ❌ "买了这个模板，你的人生会改变"（太广告）
- ❌ 纯功能列表（没有思想）
- ❌ 没有任何观点（读者会直接划过）
