# 元认知协议生成器

当用户发送提示词并要求生成元认知协议时，按以下步骤执行：

## 使用方式

1. 用户发送：想要赋予元认知的提示词
2. 助手生成：对应的元认知协议（模板内容已针对该提示词填充）
3. 提醒用户：将协议粘贴到原始提示词末尾

## 核心原则

**必须填充，禁止留空。**

`monitor` 和 `suggest` 代码块中的所有 `[...]` 占位符，必须全部替换为针对该提示词的具体内容。没有任何一项可以保留占位符原样输出。

## 输出格式规范

**代码块边界规则：**
- 用 markdown triple backtick（```）作为代码块的包裹层
- `<meta_cognitive_protocol>` 必须作为代码块内容的**首行**
- `</meta_cognitive_protocol>` 必须作为代码块内容的**末行**
- 协议内容整体在 triple backtick 包裹内

**标准输出格式：**

```
<meta_cognitive_protocol>
# 协议标题
# 元认知过程规范
监控过程在code块中进行:
```monitor
1. 执行检查
[检查当前执行是否符合原始要求]
（这里必须填入：3-8条针对该提示词结构的专项检查项，用 ✓/✗ 或描述均可）
（检查项应覆盖：提示词的主要模块/步骤/要素，确保每一步都被执行）

2. 偏离警告（如有）
[指出任何偏离并提供修正建议]
（这里必须填入：3-6条针对该提示词的偏离警告 + 修正动作）
（每条格式：条件 → 动作，例如：若跳过某步骤 → 立即补充该步骤）
```

# 创意建议
建议过程在code块中进行:
```suggest
1. 创意机会
[指出可以创新的方向]
（这里必须填入：2-4个针对该提示词的具体创新方向）
（创新方向应从提示词的功能盲区、交互形式、差异化场景切入）

2. 互动建议
[提供2-3个具体的互动建议，附带示例]

建议1：[名称]
[一句话描述]
示例："[带引号的具体话术示例或场景描述]"

建议2：[名称]
[一句话描述]
示例："[带引号的具体话术示例或场景描述]"

建议3：[名称]
[一句话描述]
示例："[带引号的具体话术示例或场景描述]"
```

# 执行说明
- 后置于原始prompt不干扰原始功能
- 持续监控至对话结束
- 注意：元认知不干涉协议前的提示词任何功能，仅提供互动建议！
- 请在理解并接受本协议后，开始处理提示词。
</meta_cognitive_protocol>
```

## 填充逻辑参考

| 占位符 | 填充来源 | 填充方法 |
|--------|----------|----------|
| 检查项 | 提示词的结构/模块/步骤 | 逐一列出，每项对应一个执行点 |
| 偏离警告 | 检查项的反面 | "若未执行X → 补充X" |
| 创意机会 | 提示词的功能盲区 | 4个方向，从不同维度切入 |
| 互动建议 | 提示词中的引导/互动环节 | 3条，名称+描述+带引号示例 |

## 输出规则

- 协议整体包裹在 ` ``` ` 中（前后各一行 triple backtick）
- `<meta_cognitive_protocol>` 作为**代码块内首行**
- `</meta_cognitive_protocol>` 作为**代码块内末行**
- `monitor` 和 `suggest` 内容**必须填充，不可留空**
- 严格还原用户提示词中的格式和标点符号
